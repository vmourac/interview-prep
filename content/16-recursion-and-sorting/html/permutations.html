Permutations

Cada nível escolhe um número ainda não usado; ao voltar, desfaz a escolha para que outro ramo comece limpo.

Controles da simulação

Raciocínio atual

Backtracking: escolher, explorar e desfazer. Sem desfazer, um ramo contamina o próximo.
ingênuagera inválidos · muitas cópias
otimizadaO(n × n!) tempo · O(n) aux.

Solução ingênua

Gerar sequências e filtrar repetições faz trabalho inútil. Outra versão copia subarrays a cada chamada, funcionando com alocação extra desnecessária.

Solução otimizada

Use path e used. Escolha um item livre, recurse, depois remova o item e marque livre de novo.

A folha acontece quando path.length === nums.length.

Big O

Tempo: O(n × n!), pois há n! respostas e copiar cada uma custa O(n).

Espaço auxiliar: O(n), sem contar a saída.

Como falar em entrevista

  • Separe estado parcial de resposta final.
  • Mostre o ponto exato do undo.
  • Diga que swaps in-place reduzem estruturas, mas não o número de respostas.

Código de referência

function permute(nums) {
  const result = [];
  const path = [];
  const used = Array(nums.length).fill(false);

  function backtrack() {
    if (path.length === nums.length) {
      result.push([...path]);
      return;
    }
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue;
      used[i] = true;
      path.push(nums[i]);
      backtrack();
      path.pop();
      used[i] = false;
    }
  }

  backtrack();
  return result;
}
