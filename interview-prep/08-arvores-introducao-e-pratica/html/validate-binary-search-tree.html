Guia sintetizado de entrevista

Validate Binary
Search Tree

Armadilha classica Checar apenas se `left < node < right` para os filhos imediatos nao basta. BST e uma propriedade global herdada de todos os ancestrais.

Hints do entrevistador

  • Um no no lado direito da raiz ainda precisa ser maior que a raiz, mesmo em profundidade.
  • Pense em limites minimo e maximo validos para cada no.
  • Outra intuicao correta: inorder de BST valida sai estritamente crescente.
      5
     / \
    1   7
       / \
      4   9

O no 4 esta no lado direito de 5.
Mesmo sendo menor que 7, ele viola o limite herdado da raiz.

Caminho ingenuo

Recalcular maximo da esquerda e minimo da direita em cada no. Funciona, mas repete exploracao de subarvores inteiras muitas vezes.

Caminho otimizado

Carregue um intervalo valido para cada no durante a DFS. Cada descida aperta um dos limites herdados.

Complexidade

Tempo ingenuo: O(n^2) no pior caso
Espaco ingenuo: O(h)
Tempo otimizado: O(n)
Espaco otimizado: O(h)

O ganho vem de eliminar retrabalho. Cada no passa a ser verificado uma unica vez com contexto suficiente para decidir localmente.

Solucao final

def is_valid_bst(root):
    def dfs(node, limite_inferior, limite_superior):
        if node is None:
            return True

        if limite_inferior is not None and node.val <= limite_inferior:
            return False
        if limite_superior is not None and node.val >= limite_superior:
            return False

        return dfs(node.left, limite_inferior, node.val) and dfs(
            node.right, node.val, limite_superior
        )

    return dfs(root, None, None)
