Topico 16 - Merge estrutural

Merge K Sorted Lists

Problema excelente para discutir por que juntar k listas uma a uma desperdiça trabalho. Esta pagina sintetiza o roteiro esperado em entrevistas a partir do tema do curso, ja que a fonte nao tem captions.

Prompt: Receba k listas ligadas ordenadas e devolva uma unica lista ordenada.

Armadilha comum

Mesclar sequencialmente parece natural, mas faz o resultado parcial ser reprocessado varias vezes. Isso pode empurrar o custo para O(Nk).

Resposta forte
  • Mescle em pares.
  • Reduza o numero de listas pela metade a cada rodada.
  • Explique por que cada no participa de O(log k) rodadas.
Complexidade

Tempo: O(N log k)

Espaco auxiliar: O(1) no merge iterativo de duas listas

O fator log vem do numero de rodadas de reducão de k listas.

Implementacao de referencia

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def mergeKLists(lists):
    if not lists:
        return None

    def merge_two(a, b):
        dummy = ListNode()
        tail = dummy

        while a and b:
            if a.val <= b.val:
                tail.next = a
                a = a.next
            else:
                tail.next = b
                b = b.next
            tail = tail.next

        tail.next = a if a else b
        return dummy.next

    while len(lists) > 1:
        merged_lists = []

        for i in range(0, len(lists), 2):
            left = lists[i]
            right = lists[i + 1] if i + 1 < len(lists) else None
            merged_lists.append(merge_two(left, right))

        lists = merged_lists

    return lists[0]
