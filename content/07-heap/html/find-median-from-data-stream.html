Find Median from Data Stream

Aqui a pergunta real nao e "como achar a mediana?". E "como preservar a mediana enquanto os dados chegam um a um, sem reordenar tudo a cada evento?"

two heaps streaming invariants
Add
O(log n)
Median
O(1)
Memory
O(n)

Prompt

Implemente uma estrutura com `addNum(num)` e `findMedian()` para um fluxo continuo de numeros.

Hints

  1. Que duas metades do conjunto precisam ser mantidas separadas?
  2. Qual heap deve guardar a metade inferior e qual deve guardar a superior?
  3. Quais invariantes precisam ser verdadeiros depois de toda insercao?

Naive Path

Guardar tudo e ordenar de novo a cada insercao. Simples, mas inviavel para um stream longo porque cada atualizacao reaproveita mal o trabalho anterior.

Optimized Path

Use uma max-heap para a metade inferior e uma min-heap para a metade superior. Rebalanceie apos cada insercao para manter tamanhos quase iguais e fronteira ordenada entre metades.

Complexity

  • Tempo de add: O(log n), por causa de um numero constante de `push` e `pop` em heaps.
  • Tempo de median: O(1), porque a resposta esta nas raizes.
  • Entrevista: a resposta forte explicita invariantes. Sem isso, parece memorizacao de biblioteca.

Final Solution

import heapq


class MedianFinder:
    def __init__(self) -> None:
        self.lower: list[int] = []
        self.upper: list[int] = []

    def addNum(self, num: int) -> None:
        heapq.heappush(self.lower, -num)
        heapq.heappush(self.upper, -heapq.heappop(self.lower))

        if len(self.upper) > len(self.lower):
            heapq.heappush(self.lower, -heapq.heappop(self.upper))

    def findMedian(self) -> float:
        if len(self.lower) > len(self.upper):
            return float(-self.lower[0])

        return (-self.lower[0] + self.upper[0]) / 2.0
