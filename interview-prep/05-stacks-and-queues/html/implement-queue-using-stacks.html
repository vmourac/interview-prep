Topico 05 · Abstracao versus implementacao

Implement Queue Using Stacks

Prompt: Construa uma queue com duas stacks e explique por que `pop` e `peek` podem ser O(1) amortizado.

Este e o tipo de problema que mostra se voce so decorou nomes de estruturas ou se realmente entende como LIFO pode ser rearranjado para produzir FIFO.

Hints

  • Separe a pilha que recebe entradas da pilha que serve saidas.
  • Evite mover elementos a cada operacao; mova apenas quando necessario.
  • A prova de complexidade depende de contar quantas vezes cada elemento troca de pilha.

Caminho ingenuo

Mover tudo de ida e volta em cada remocao funciona, mas desperdiça trabalho. Voce paga custo linear repetidamente mesmo quando nao precisava.

Caminho otimizado

Use `in_stack` para receber elementos e `out_stack` para expor a frente da fila. A transferencia so acontece quando `out_stack` esta vazia.

Complexidade

Push: O(1)

Pop/Peek: O(1) amortizado

Espaco: O(n)

Solucao final

class MyQueue:
    def __init__(self):
        self.in_stack = []
        self.out_stack = []

    def _move_if_needed(self):
        if self.out_stack:
            return

        while self.in_stack:
            self.out_stack.append(self.in_stack.pop())

    def push(self, x):
        self.in_stack.append(x)

    def pop(self):
        self._move_if_needed()
        return self.out_stack.pop()

    def peek(self):
        self._move_if_needed()
        return self.out_stack[-1]

    def empty(self):
        return not self.in_stack and not self.out_stack
