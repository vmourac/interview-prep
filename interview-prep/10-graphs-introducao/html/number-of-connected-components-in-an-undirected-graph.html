Transit Map Drill

Number of Connected Components in an Undirected Graph

Este prompt mede se voce sabe decompor um grafo em blocos desconectados e contar cada travessia nova como um componente.

Nota de fonte: este material e sintetizado a partir do topico do curso, porque a aula original nao tem captions nem transcript disponiveis.

Graph Build: O(E)
Traversal: O(V + E)
Aux Space: O(V + E)
Core Idea: unvisited node starts a new component

Prompt

Dado um grafo nao direcionado com `n` vertices e uma lista `edges`, retorne quantos componentes conectados existem.

Why This Matters

O entrevistador quer ver se voce traduz conectividade em travessias completas e se sabe justificar `O(V + E)` sem decorar formula.

Naive Route

Reexaminar a lista de arestas varias vezes para descobrir quem se conecta com quem. Funciona mal porque repete checagens que a lista de adjacencia resolveria de forma natural.

Optimized Route

Monte a lista de adjacencia e inicie DFS ou BFS apenas quando encontrar um vertice ainda nao visitado. Cada inicio desses conta um componente novo.

Reasoning Steps

  1. Construa o grafo nos dois sentidos.
  2. Passe por `0` ate `n - 1`.
  3. Se o no ja foi visitado, ignore.
  4. Se nao foi, incremente o contador e explore todo o bloco conectado.
  5. No fim, o contador e o numero de componentes.

Final Solution

from collections import defaultdict


class Solution:
    def countComponents(self, n: int, edges: list[list[int]]) -> int:
        graph = defaultdict(list)

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        visited = set()
        components = 0

        def dfs(node: int) -> None:
            visited.add(node)

            for neighbor in graph[node]:
                if neighbor not in visited:
                    dfs(neighbor)

        for node in range(n):
            if node in visited:
                continue

            components += 1
            dfs(node)

        return components
