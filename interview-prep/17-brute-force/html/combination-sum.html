Combination Sum

A busca permite reutilizar números, mas controla a ordem com start para evitar permutações duplicadas da mesma combinação.

Controles da simulação

Raciocínio atual

Pruning seguro: com candidatos positivos e ordenados, se value > remaining, os próximos também excedem.
ingênuaexponencial + dedupe
otimizadaO(n^d · d) tempo · O(d) DFS

Solução ingênua

Tentar qualquer candidato em qualquer ordem gera sequências equivalentes como [2,3,2] e [2,2,3], exigindo deduplicação depois.

Solução otimizada

Ordene, carregue remaining e recurse a partir de start. Ao escolher um valor, chame DFS com o mesmo índice porque repetição é permitida.

Quando remaining === 0, copie o caminho.

Big O

Defina n = candidates.length, target = T, m = min(candidates) e d = floor(T / m), assumindo candidatos positivos.

Tempo: O(n log n + n^d · d) é um limite superior: há no máximo n escolhas por nível, profundidade máxima d e até O(d) para copiar uma solução. Sem introduzir d, o termo da busca pode ser escrito como O(n^(target / m) · target / m).

Espaço: O(d) auxiliar para stack e path, sem contar os detalhes internos do sort; com S soluções, a saída ocupa O(S · d).

Como falar em entrevista

  • Defina n, T, m e a profundidade d.
  • Diga que start evita permutações duplicadas.
  • Explique por que a poda é segura e separe melhoria prática do limite superior exponencial.

Código de referência

function combinationSum(candidates, target) {
  candidates.sort((a, b) => a - b);
  const result = [];
  const path = [];

  function dfs(start, remaining) {
    if (remaining === 0) {
      result.push([...path]);
      return;
    }
    for (let i = start; i < candidates.length; i++) {
      const value = candidates[i];
      if (value > remaining) break;
      path.push(value);
      dfs(i, remaining - value);
      path.pop();
    }
  }

  dfs(0, target);
  return result;
}
