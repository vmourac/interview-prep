Recursion to Iteration

Climbing Stairs

Uma pergunta pequena com uma bifurcacao importante: se voce seguir pela recursao pura, cai em recomputacao exponencial. Se enxergar a repeticao de estados, transforma o problema em programacao dinamica enxuta.

Prompt

Voce pode subir 1 ou 2 degraus por vez. Quantas formas distintas existem para chegar exatamente ao degrau n?

Hints

  1. Quais foram os ultimos passos possiveis antes de chegar ao topo?
  2. Quantas vezes a recursao pura recalcula o mesmo estado?
  3. Voce realmente precisa guardar a tabela inteira?

Naive Path

Defina ways(n) = ways(n - 1) + ways(n - 2) e implemente com recursao pura. A relacao esta certa, mas a arvore de chamadas repete muitos subproblemas, como no Fibonacci classico.

Optimized Path

Calcule as respostas de baixo para cima. Como cada estado depende so dos dois anteriores, use duas variaveis para carregar o historico minimo necessario.

Complexity

  • Tempo: O(n), com um calculo por degrau a partir do terceiro.
  • Espaco: O(1), porque so os dois estados anteriores sao mantidos.
  • Entrevista: conecte a versao ingenua ao padrao exponencial visto no curso.

Final Solution

def climbStairs(n: int) -> int:
    if n <= 2:
        return n

    prev2 = 1
    prev1 = 2

    for _ in range(3, n + 1):
        current = prev1 + prev2
        prev2 = prev1
        prev1 = current

    return prev1
