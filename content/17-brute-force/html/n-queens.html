N-Queens

A busca ainda é exponencial, mas o algoritmo forte poda conflitos no instante da escolha usando colunas e diagonais ocupadas.

Controles da simulação

Raciocínio atual

Diagonais: casas na mesma diagonal principal compartilham row - col; na secundária, row + col.
ingênuaO(n^n) + validação
otimizada~O(n!) tempo · O(n²) espaço

Solução ingênua

Escolher uma coluna por linha sem checar conflitos durante a construção gera muitos tabuleiros inválidos, validados tarde demais.

Solução otimizada

Busque linha por linha. Antes de posicionar, cheque coluna, diagonal row - col e diagonal row + col em O(1).

Ao voltar, remova a rainha e desmarque os três conjuntos.

Big O

Tempo: aproximadamente O(n!) no pior caso.

Espaço: O(n²) para o tabuleiro, mais O(n) para stack/conjuntos, sem contar a saída.

Como falar em entrevista

  • Explique uma rainha por linha.
  • Justifique as fórmulas das diagonais.
  • Destaque que não revalida o tabuleiro inteiro.

Código de referência

function solveNQueens(n) {
  const result = [];
  const board = Array.from({ length: n }, () => Array(n).fill("."));
  const cols = new Set();
  const diag1 = new Set();
  const diag2 = new Set();

  function dfs(row) {
    if (row === n) {
      result.push(board.map((line) => line.join("")));
      return;
    }
    for (let col = 0; col < n; col++) {
      if (cols.has(col) || diag1.has(row - col) || diag2.has(row + col)) continue;
      board[row][col] = "Q";
      cols.add(col); diag1.add(row - col); diag2.add(row + col);
      dfs(row + 1);
      board[row][col] = ".";
      cols.delete(col); diag1.delete(row - col); diag2.delete(row + col);
    }
  }

  dfs(0);
  return result;
}
