Climbing Stairs

A recorrência é a mesma de Fibonacci: para chegar ao degrau n, o último movimento veio de n - 1 ou n - 2.

Controles da simulação

Raciocínio atual

ingênuaO(2ⁿ) tempo · O(n) espaço

Solução ingênua

Use recursão pura: ways(n) = ways(n - 1) + ways(n - 2). Correta, mas recalcula os mesmos estados muitas vezes.

  • Tempo: O(2^n).
  • Espaço: O(n) pela pilha.

Solução otimizada

Calcule de baixo para cima e mantenha só os dois últimos resultados: prev2 e prev1.

Invariante: antes de calcular o degrau atual, prev2 é ways(i-2) e prev1 é ways(i-1).

Big O

Tempo otimizado: O(n), cada degrau é calculado uma vez.

Espaço otimizado: O(1), apenas variáveis escalares.

Como falar em entrevista

  • Relacione com Fibonacci para explicar a árvore explosiva.
  • Mostre os casos base n = 1 e n = 2.
  • Justifique por que uma tabela completa é desnecessária.
  • Finalize com a versão iterativa.

Código de exemplo

function climbStairs(n) {
  if (n <= 2) return n;
  let prev2 = 1;
  let prev1 = 2;

  for (let step = 3; step <= n; step += 1) {
    const current = prev1 + prev2;
    prev2 = prev1;
    prev1 = current;
  }

  return prev1;
}
