Top K Frequent Words

Aqui heap sozinha nao basta. O entrevistador quer ver se voce combina contagem, ranking parcial e desempate lexicografico sem cair na tentacao de ordenar tudo.

Prompt

Dado um array de palavras e um inteiro `k`, devolva as `k` palavras mais frequentes. Empates de frequencia devem respeitar ordem lexicografica crescente.

Hints

  1. Qual parte do problema e so dicionario?
  2. Como representar "pior candidato atual" em uma heap de tamanho fixo?
  3. Por que talvez valha ordenar so os `k` sobreviventes no fim?

Naive Path

Conte tudo e ordene todas as palavras distintas pela regra final do problema. Correto, mas gasta `log m` sobre todo o universo distinto.

Optimized Path

Conte frequencias e mantenha so os `k` melhores em uma min-heap. O topo representa o pior entre os escolhidos; quando aparece algo melhor, ele sai.

Complexity

  • Tempo: O(n + m log k + k log k).
  • Espaco: O(m + k).
  • Entrevista: explique com clareza o desempate lexicografico. Esse e o ponto que mais derruba respostas mecanicas.

Final Solution

import heapq
from collections import Counter


class Entry:
    def __init__(self, word: str, count: int) -> None:
        self.word = word
        self.count = count

    def __lt__(self, other: "Entry") -> bool:
        if self.count == other.count:
            return self.word > other.word
        return self.count < other.count


def topKFrequent(words: list[str], k: int) -> list[str]:
    freq = Counter(words)
    heap: list[Entry] = []

    for word, count in freq.items():
        heapq.heappush(heap, Entry(word, count))
        if len(heap) > k:
            heapq.heappop(heap)

    result = []
    while heap:
        result.append(heapq.heappop(heap))

    result.sort(key=lambda entry: (-entry.count, entry.word))
    return [entry.word for entry in result]
