Cycle Trigger

Redundant Connection

O momento importante e o instante em que uma aresta tenta conectar dois vertices que ja estavam ligados por algum caminho.

1-2
2-3
1-3
Signal: primeira uniao que fecha ciclo
Tempo: O(E * alpha(V))
Espaco: O(V)
Ferramenta: Union-Find incremental

Prompt

Uma arvore recebeu uma aresta extra. Descubra qual aresta e a redundante.

Hints

  1. Se ja existe caminho entre `u` e `v`, a nova aresta fecha um ciclo.
  2. Voce nao precisa guardar o grafo inteiro para responder.
  3. Uma falha de `union` ja encerra o problema.

Naive Path

Adicionar arestas e rodar DFS a cada passo repete muita exploracao e esconde a estrutura incremental do problema.

Optimized Path

Use Union-Find. Se os dois extremos ja tem o mesmo representante, a aresta atual e a redundante.

Interview Note

O entrevistador quer ouvir a traducao semantica correta: mesmo representante significa componente ja conectada, logo nova aresta implica ciclo.

Final Solution

def findRedundantConnection(edges: list[list[int]]) -> list[int]:
    n = len(edges)
    parent = list(range(n + 1))
    rank = [0] * (n + 1)

    def find(node: int) -> int:
        if parent[node] != node:
            parent[node] = find(parent[node])
        return parent[node]

    def union(a: int, b: int) -> bool:
        root_a = find(a)
        root_b = find(b)

        if root_a == root_b:
            return False

        if rank[root_a] < rank[root_b]:
            parent[root_a] = root_b
        elif rank[root_a] > rank[root_b]:
            parent[root_b] = root_a
        else:
            parent[root_b] = root_a
            rank[root_a] += 1

        return True

    for u, v in edges:
        if not union(u, v):
            return [u, v]

    return []
