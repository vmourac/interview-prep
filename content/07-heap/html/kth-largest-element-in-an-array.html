Control Room / Heap Filter

Kth Largest Element in an Array

O entrevistador nao quer saber se voce consegue ordenar. Ele quer saber se voce percebe que a pergunta pede filtragem incremental: preservar apenas os `k` maiores enquanto o restante pode ser descartado.

min-heap top-k selection
Naive
O(n log n)
Optimized
O(n log k)
Memory
O(k)

Prompt

Dado `nums` e `k`, retorne o `k`-esimo maior valor do array. Nao e o `k`-esimo distinto; e a posicao na ordem classificada.

Hints

  1. Voce precisa mesmo da ordem completa dos `n` elementos?
  2. Que estrutura mantem o menor entre os `k` melhores sempre visivel?
  3. O que acontece se a heap ultrapassar tamanho `k`?

Naive Path

Ordene tudo em ordem decrescente e leia `nums[k - 1]`. Funciona, mas paga custo global para responder uma pergunta local.

Optimized Path

Mantenha uma min-heap de tamanho `k`. Quando ela cresce demais, remova o menor. No fim, a raiz da heap e o `k`-esimo maior global.

Complexity

  • Tempo: O(n log k), porque cada elemento gera no maximo operacoes de heap sobre tamanho controlado `k`.
  • Espaco: O(k), porque a heap nunca armazena mais que `k` elementos.
  • Entrevista: a frase forte e "eu nao preciso do array ordenado; eu so preciso preservar a fronteira dos `k` maiores".

Final Solution

import heapq


def findKthLargest(nums: list[int], k: int) -> int:
    heap: list[int] = []

    for value in nums:
        heapq.heappush(heap, value)
        if len(heap) > k:
            heapq.heappop(heap)

    return heap[0]
