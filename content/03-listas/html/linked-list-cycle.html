Linked List Cycle

Pagina local e autocontida para revisar o padrao de slow e fast pointers. O conteudo foi sintetizado do tema do curso, pois as aulas do topico nao fornecem captions para uma compilacao textual direta.

Prompt: Dado o head de uma lista ligada simples, determine se existe ciclo.

Hints

  • Existe uma solucao com conjunto de visitados, mas o entrevistador costuma querer menos memoria.
  • Dois ponteiros com velocidades diferentes revelam estruturas ciclicas.
  • Se o ponteiro rapido nunca encontra null, pergunte-se onde ele pode encontrar o lento.

Caminho ingenuo

Guardar todos os nos visitados em um conjunto detecta repeticao rapidamente, mas usa memoria proporcional ao numero de nos percorridos.

Caminho otimizado

Floyd usa dois ponteiros. O lento anda um passo, o rapido anda dois. Se houver ciclo, a diferenca de velocidade faz um ponteiro alcancar o outro dentro do laco.

Complexidade

Tempo: O(n)

Espaco: O(1)

A memoria fica constante porque a deteccao depende apenas de dois cursores, nao de registrar historico.

Solucao final

def hasCycle(head):
    slow = head
    fast = head

    while fast is not None and fast.next is not None:
        slow = slow.next
        fast = fast.next.next

        if slow is fast:
            return True

    return False
