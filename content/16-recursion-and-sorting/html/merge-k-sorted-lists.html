Solução ingênua

Mesclar listas uma por uma faz a lista parcial crescer e ser percorrida de novo muitas vezes, podendo custar O(Nk).

Solução otimizada

Mescle pares em rodadas. A quantidade de listas cai aproximadamente pela metade até sobrar uma lista final.

Cada nó participa de um merge por rodada, e há O(log k) rodadas.

Big O

Tempo: O(N log k), com N nós totais.

Espaço auxiliar: O(k) para o array de referências da maior rodada; cada merge de duas listas usa O(1) adicional e reutiliza os nós.

Como falar em entrevista

  • Defina N e k.
  • Explique por que merge sequencial reprocessa nós.
  • Mostre a analogia com merge sort e contabilize o array de referências.

Código de referência

function mergeKLists(lists) {
  // Espaço auxiliar: O(k) para os arrays de referências das rodadas.
  if (lists.length === 0) return null;
  while (lists.length > 1) {
    const merged = [];
    for (let i = 0; i < lists.length; i += 2) {
      merged.push(mergeTwo(lists[i], lists[i + 1] ?? null));
    }
    lists = merged;
  }
  return lists[0];
}

function mergeTwo(a, b) {
  const dummy = { val: 0, next: null };
  let tail = dummy;
  while (a && b) {
    if (a.val <= b.val) { tail.next = a; a = a.next; }
    else { tail.next = b; b = b.next; }
    tail = tail.next;
  }
  tail.next = a ?? b;
  return dummy.next;
}
type ListNode = { val: number; next: ListNode | null };
