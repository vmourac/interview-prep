Group Anagrams

Controles da simulação

Raciocínio atual

Atenção: A chave precisa preservar frequência de letras; apenas usar Set de letras confundiria, por exemplo, ab e aab.
ingênuaO(n²·k log k) tempo · O(n·k) espaço
otimizadaO(n·k log k) tempo · O(n·k) espaço

Solução ingênua

Compare cada palavra com grupos existentes verificando anagrama par a par. Isso mistura agrupamento com comparação repetida. Os grupos armazenam O(n·k) caracteres no total; a comparação pode usar O(k) temporário.

Solução otimizada

Ordene os caracteres da palavra para criar a chave canônica. Palavras com a mesma chave entram no mesmo bucket do Map.

Ideia-chave: mantenha um estado pequeno e reutilizável em vez de reprocessar o histórico.

Big O

Para n palavras de tamanho médio k, a solução ingênua custa O(n²·k log k) tempo e O(n·k) espaço para os grupos. Na otimizada, cada chave ordenada custa O(k log k), feita uma vez por palavra: O(n·k log k) tempo e O(n·k) espaço para chaves, mapa e grupos.

Como falar em entrevista

Destaque a ideia principal: converter equivalência semântica em lookup O(1) médio por chave.

Código de referência

function groupAnagrams(strs) {
  const groups = new Map();

  for (const word of strs) {
    const key = word.split('').sort().join('');
    if (!groups.has(key)) groups.set(key, []);
    groups.get(key).push(word);
  }

  return Array.from(groups.values());
}
function groupAnagrams(strs: string[]): string[][] {
  const groups = new Map<string, string[]>();

  for (const word of strs) {
    const key = word.split('').sort().join('');
    if (!groups.has(key)) groups.set(key, []);
    groups.get(key)!.push(word);
  }

  return Array.from(groups.values());
}
