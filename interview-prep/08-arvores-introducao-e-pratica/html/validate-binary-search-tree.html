Validate Binary Search Tree

BST válido exige limites herdados de todos os ancestrais, não apenas comparação local com filhos imediatos.

Raciocínio atual

Foco de entrevista: O erro clássico é checar só filho esquerdo < nó < filho direito; isso deixa violações profundas passarem.
ingênuaO(n²) tempo · O(h) espaço
otimizadaO(n) tempo · O(h) espaço

Solução ingênua

Para cada nó, recalcule máximo da esquerda e mínimo da direita. Funciona, mas repete trabalho em subárvores.

  • Cada verificação pode varrer subárvores.
  • No pior caso vira O(n²).

Solução otimizada

Empilhe cada nó com seu intervalo permitido. A esquerda recebe novo limite superior; a direita, novo limite inferior.

Invariante: cada item da pilha mantém os limites herdados do caminho desde a raiz.

Big O

Cada nó faz comparações constantes uma vez: O(n). A pilha explícita custa O(h).

Como falar em entrevista

  • Use limites abertos, não inclusivos.
  • Explique por que duplicatas violam se o problema pede BST estrita.
  • Mencione inorder crescente como alternativa.

Código de referência

function isValidBST(root) {
  const stack = [[root, null, null]];
  while (stack.length) {
    const [node, low, high] = stack.pop();
    if (node === null) continue;
    if (low !== null && node.val <= low) return false;
    if (high !== null && node.val >= high) return false;
    stack.push([node.right, node.val, high]);
    stack.push([node.left, low, node.val]);
  }
  return true;
}
type TreeNode = { val: number; left: TreeNode | null; right: TreeNode | null };
