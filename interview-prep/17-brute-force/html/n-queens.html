Este e o retrato classico de brute force com pruning pesado: nao basta tentar casas, e preciso manter colunas e diagonais ocupadas para matar ramos cedo.

Signal: constraint search
Tempo: aproximadamente O(n!)
Espaco: O(n^2) + stack
Chave: colunas e diagonais

Prompt

Posicione `n` rainhas em um tabuleiro `n x n` sem ataques por linha, coluna ou diagonal, retornando todas as configuracoes validas.

Hints

  1. Uma rainha por linha reduz o estado.
  2. Cheque conflito em O(1) com conjuntos.
  3. Podar cedo importa mais do que validar tarde.

Naive Path

Preencher o tabuleiro inteiro e so depois testar ataques gera um mar de estados inutilmente invalidos.

Optimized Path

Busque linha por linha, colocando uma rainha apenas em colunas e diagonais ainda livres. Cada conflito conhecido elimina o ramo no mesmo instante.

Interview Note

O entrevistador quer ouvir que `row - col` e `row + col` codificam diagonais e tornam a validacao incremental em O(1).

Final Solution

def solveNQueens(n: int) -> list[list[str]]:
    result = []
    board = [["."] * n for _ in range(n)]
    cols = set()
    diag1 = set()
    diag2 = set()

    def dfs(row: int) -> None:
        if row == n:
            result.append(["".join(line) for line in board])
            return

        for col in range(n):
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue

            board[row][col] = "Q"
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)

            dfs(row + 1)

            board[row][col] = "."
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    dfs(0)
    return result
